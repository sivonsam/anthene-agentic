🔄 Datavirta — agentin suoritus
1. Käyttäjä kirjoittaa kysymyksen Prophet/Store -näkymässä  → 2. POST /api/run/{agent_id} (Bearer JWT)  → 3. LangGraph StateGraph käynnistyy  → 4. GPT-4o päättää seuraavan tool-kutsun  → 5. Tool Hub hakee datan (ADS-B / sää / jne.)  → 6. GPT-4o analysoi tulokset, päättää jatkoaskeleet  → 7. SSE stream palauttaa tokenit + tool-tapahtumat reaaliajassa  → 8. Frontend renderöi vastauksen streamattuna

Karttavirta (tool results → kartta): Frontend POST /api/tools/call/adsb_area → API → Tool Hub → ADS-B Exchange → GeoJSON features → MapLibre GL renderöi kartalle

Lentokone-seuranta: "Seuraa kartalla" → POST /api/tools/call/adsb_by_callsign (30s välein) → sijainti päivitetään kartalle, sininen piste + trail-viiva, kartta seuraa automaattisesti

Telegram-hälytysvirta: Agentti kutsuu telegram_alert-työkalua → Tool Hub → Telegram Bot API → @AntheneAgenticBot → henkilökohtainen kanava (chat_id per käyttäjä) → executor tallentaa hälytyksen myös Cosmos DB:hen → Alerts UI näyttää sen

Suora seurantalinkki (v2.6): https://green-glacier-0ec1a7c03.7.azurestaticapps.net?track=CALLSIGN → Prophet avautuu suoraan karttanäkymään → automaattinen seuranta callsignilla tai rekisteritunnuksella → 5s päivitysväli, sininen pulssimarkkeri kartalla

ADD_TOOL chat-nappi (v2.6): Agentti vastaa [ADD_TOOL: tool_id | Nimi] → TestChat renderöi klikkattavan ➕-napin → nappia painamalla työkalu lisätään agenttiin suoraan API PATCH -kutsulla ilman Creator-sivulle siirtymistä

🔐 Pääsynhallinta
Demo-autentikaatio (DEV_MODE) — Kun Azure AD B2C ei ole konfiguroitu, käytetään paikallista JWT-kirjautumista.
Frontend näyttää DevLogin-lomakkeen → POST /api/auth/login → palautetaan HS256 JWT (8h) → tallennetaan localStorage:een → lähetetään Authorization: Bearer -headerissa.
API validoi ensin HS256-tokenit (issuer: anthene-local), sitten B2C RS256 -tokenit. JWT_SECRET = env var JWT_SECRET (oletus: anthene-demo-secret-2025).
